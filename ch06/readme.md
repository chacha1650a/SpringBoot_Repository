## 1. 문서 개요

### 1.1 목적
본 문서는 계정별 할 일(TODO) 관리 웹 서비스의 기능 및 비기능 요구사항을 정의한다. 개발팀(수강생)이 동일한 목표를 공유하고, 구현 범위와 완료 기준을 명확히 하는 것을 목적으로 한다.

### 1.2 프로젝트 개요
Apple iOS의 "미리알림(Reminders)" 앱을 벤치마킹하여, 사용자가 OAuth2 소셜 로그인으로 가입하고 카테고리(목록)별·날짜별로 할 일을 관리할 수 있는 서비스를 구현한다.

### 1.3 범위 (Scope)
| 구분 | 포함 | 제외 |
| --- | --- | --- |
| MVP (필수) | 소셜 로그인, TODO CRUD, 카테고리, 캘린더(월/일), 완료 처리 | 팀 공유, 결제 |
| 확장 (선택) | 브라우저 푸시 알림, 반복 일정, 하위 작업, 검색 | 모바일 네이티브 앱 |

## 2. 시스템 구성

### 2.1 기술 스택 (권장)
| 영역 | 기술 |
| --- | --- |
| Frontend | React 18, Zustand(상태관리), React Router, Axios |
| Backend | Spring Boot 3.x, Spring Security 6, Spring Data JPA |
| 인증 | OAuth2 Client + JWT (Access / Refresh Token) |
| DB | MySQL 8.x |
| 빌드/배포 | Gradle, Docker (선택) |

### 2.2 시스템 아키텍처
```
[React SPA] --(REST API / JWT)--> [Spring Boot] --(JPA)--> [MySQL]
     |
     └--(OAuth2 redirect)--> [Google / Naver / Kakao 인증 서버]
```

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 인증 / 계정 관리

| ID | 요구사항 | 우선순위 |
| --- | --- | --- |
| FR-AUTH-01 | 사용자는 Google 계정으로 로그인할 수 있다. | 필수 |
| FR-AUTH-02 | 사용자는 Naver 계정으로 로그인할 수 있다. | 필수 |
| FR-AUTH-03 | 사용자는 Kakao 계정으로 로그인할 수 있다. | 필수 |
| FR-AUTH-04 | 최초 로그인 시 사용자 정보(이메일, 닉네임, 프로필 이미지, provider)를 DB에 저장한다. | 필수 |
| FR-AUTH-05 | 이미 가입된 사용자는 동일 provider 재로그인 시 기존 계정으로 인증된다. | 필수 |
| FR-AUTH-06 | 로그인 성공 시 서버는 JWT(Access Token)를 발급하고 클라이언트는 이를 저장한다. | 필수 |
| FR-AUTH-07 | 모든 데이터 API 요청은 JWT로 인증된 사용자 본인의 데이터에만 접근할 수 있다. | 필수 |
| FR-AUTH-08 | 사용자는 로그아웃할 수 있으며, 토큰은 폐기/만료 처리된다. | 필수 |
| FR-AUTH-09 | Access Token 만료 시 Refresh Token으로 재발급받을 수 있다. | 선택 |

> **보안 핵심**: 모든 TODO/카테고리 조회·수정·삭제는 토큰에서 추출한 사용자 식별자(userId) 기준으로 필터링한다. 타인의 데이터에 접근하면 403을 반환한다.

### 4.2 카테고리 (목록) 관리
> iOS 미리알림의 "목록(List)" 개념에 해당한다.

| ID | 요구사항 | 우선순위 |
| --- | --- | --- |
| FR-CAT-01 | 사용자는 카테고리를 생성할 수 있다. (이름 필수) | 필수 |
| FR-CAT-02 | 사용자는 카테고리에 색상을 지정할 수 있다. | 필수 |
| FR-CAT-03 | 사용자는 카테고리에 아이콘(이모지 등)을 지정할 수 있다. | 선택 |
| FR-CAT-04 | 사용자는 카테고리 이름/색상을 수정할 수 있다. | 필수 |
| FR-CAT-05 | 사용자는 카테고리를 삭제할 수 있다. (소속 TODO 처리 정책 정의 필요) | 필수 |
| FR-CAT-06 | 카테고리별 미완료 할 일 개수가 표시된다. | 필수 |
| FR-CAT-07 | 가입 시 기본 카테고리("미분류" 등)가 자동 생성된다. | 선택 |

> **삭제 정책**: 카테고리 삭제 시 ① 소속 TODO도 함께 삭제 또는 ② "미분류"로 이동 중 택일하여 명세한다. (팀 결정 사항)

## 6. 데이터 모델 (개념 ERD)

```
User (사용자)
 ├─ id (PK)
 ├─ email
 ├─ nickname
 ├─ profile_image
 ├─ provider (GOOGLE / NAVER / KAKAO)
 ├─ provider_id
 └─ created_at

Category (카테고리)
 ├─ id (PK)
 ├─ user_id (FK → User)
 ├─ name
 ├─ color
 ├─ icon
 └─ created_at

Todo (할 일)
 ├─ id (PK)
 ├─ user_id (FK → User)
 ├─ category_id (FK → Category, nullable)
 ├─ title
 ├─ memo
 ├─ due_date
 ├─ due_time
 ├─ priority
 ├─ is_flagged
 ├─ is_completed
 ├─ completed_at
 ├─ created_at
 └─ updated_at
```

**관계**
- User(1) : Category(N)
- User(1) : Todo(N)
- Category(1) : Todo(N)

## 7. API 명세 (요약)

| 기능 | Method | URL |
| --- | --- | --- |
| 소셜 로그인 진입 | GET | `/oauth2/authorization/{provider}` |
| 내 정보 조회 | GET | `/api/users/me` |
| 카테고리 목록 | GET | `/api/categories` |
| 카테고리 생성 | POST | `/api/categories` |
| 카테고리 수정 | PUT | `/api/categories/{id}` |
| 카테고리 삭제 | DELETE | `/api/categories/{id}` |
| 할 일 목록(필터) | GET | `/api/todos?date=&categoryId=&completed=` |
| 할 일 생성 | POST | `/api/todos` |
| 할 일 수정 | PUT | `/api/todos/{id}` |
| 할 일 완료 토글 | PATCH | `/api/todos/{id}/complete` |
| 할 일 삭제 | DELETE | `/api/todos/{id}` |

> 모든 `/api/**` 요청은 `Authorization: Bearer {accessToken}` 헤더를 요구한다.
